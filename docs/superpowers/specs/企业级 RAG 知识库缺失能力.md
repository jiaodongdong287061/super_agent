# 企业级 RAG 知识库缺失能力

当前系统已完成：多格式加载、向量化、语义检索、混合检索+Reranker、标签体系、增量索引、向量库可切换。

以下为缺失的关键能力：

## 1. LLM 答案生成（最关键）
当前 `/rag/query` 只拼接 chunk 原文返回，没有 LLM 生成环节。企业用户要的是答案，不是原文片段。需要接入 LLM 做 RAG 生成，带引用溯源。

## 2. 权限控制
无文档级/部门级访问控制。需要根据用户身份过滤可检索的文档范围。

## 3. 多租户
不同业务线数据隔离。需要集合级或命名空间级的隔离机制。

## 4. 审计追踪
无查询日志。需要记录谁查了什么、何时查的，满足合规要求。

## 5. 查询理解
原文直接 embed，无 Query 改写/扩展/意图识别。需要 LLM 辅助的查询预处理。

## 6. 语义分块增强
当前为规则切分（标题+滑动窗口）。需要 LLM 辅助的真正语义分块。

## 7. 可观测性
只有基础日志。需要检索命中率、延迟、召回率等指标，以及 OpenTelemetry 集成。

## 8. 文档管理
无版本管理、过期淘汰、更新通知。需要文档生命周期管理能力。
